# Provenance Website Strategy + UX + Content + SEO Specification

This document is the single source of truth for the Provenance marketing site. It defines positioning, audience, pages, copy direction, visual rules, SEO requirements, technical implementation, and launch expectations so design and build decisions stay aligned.

## 1. Brand and positioning

- Company name usage: Provenance
- One-line description: Audit-ready decision traceability for AI-assisted financial workflows.
- Longer positioning statement: Provenance is the decision-trace layer for AI-assisted and exception-heavy financial workflows. It helps teams reconstruct, capture, review, and export the reasoning behind important decisions in an audit-ready format.
- Target audience: risk, compliance, operations, controls, AI governance, model risk, internal tooling stakeholders, accelerator reviewers, investor reviewers, and potential design partners.
- Core promise: turn fragmented evidence into reviewable decision dossiers that make high-stakes workflows inspectable, governable, and export-ready.
- Primary CTA: Request Demo

Messaging anchor:

> Audit-ready decision traceability for AI-assisted and exception-heavy financial workflows.

Secondary one-liners to reuse:

- The decision context layer for financial services.
- Turn fragmented evidence into reviewable decision dossiers.
- Capture the why behind high-stakes decisions.
- Make AI-assisted workflows traceable, reviewable, and regulator-ready.

## 2. Audience and conversion goals

## 5. Design principles

Core principles:

- Precision over hype.
- Structured like a control system.
- Editorial plus systems aesthetic.
- Motion only where meaning exists.
- Dark, intelligent, high-contrast by default.
- Distinctive visual motif built from trace lines, evidence nodes, casefile layers, and timeline stitching.

Explicitly avoid:

- Generic AI glows.
- Floating cubes.
- Random neon gradients.
- Startup-template testimonial carousels.
- Stock illustration people.
- Cyberpunk decoration.

Desired reaction:

> This feels like financial infrastructure with product taste, not another AI startup template.

## 6. Visual system spec

Typography:

- Two-font system.
- Body: clean, modern, highly legible sans-serif.
- Accent/display: elegant serif or semi-serif used sparingly for section titles or highlighted lines.
- Strong hierarchy, generous spacing, and expensive-looking typography.

Spacing and grid:

- Generous whitespace with modular spacing steps.
- 12-column desktop grid with intentional asymmetry.
- Comfortable mobile stacking with preserved hierarchy.

Color:

- Background: charcoal, graphite, and black-adjacent surfaces.
- Text: soft off-white with muted cool-gray secondary text.
- Accent: one restrained intelligent blue/teal family.
- Accent use limited to traces, highlights, key tags, and CTAs.

Component patterns:

- Thin evidence-line dividers.
- Layered cards with subtle inner highlights.
- Dossier preview pattern with metadata, evidence, reasoning, policy, and review regions.
- Timeline stitcher pattern with approvals, notes, policies, and state changes.
- Screenshot framing style that feels like a product specimen, not a browser mockup.
- Icon system based on nodes, links, files, policy markers, approvals, timelines, exports, and review states.

Motion rules:

- Use lightweight motion to reveal evidence linking, dossier assembly, confidence shifts, or timeline progression.
- No decorative floating motion.
- No heavy parallax.

## 8. SEO and discoverability spec

Required at launch:

- Google Search Console verification
- Bing Webmaster Tools verification
- IndexNow setup
- Valid `sitemap.xml`
- Valid `robots.txt`
- Metadata for every public page
- Open Graph tags
- Canonical URLs
- Favicon and social preview image
- Mobile QA
- Performance QA
- Analytics and event tracking

Important constraints:

- `robots.txt` controls crawler access but does not guarantee de-indexing.
- Use `noindex` or access controls for pages that should stay out of search.
- Sitemaps should list all important public URLs.
- Metadata must be written per page, not treated as an afterthought.

Ongoing process:

- Publish structured thought leadership or product insights after launch.
- Monitor indexing, branded queries, and page coverage.
- Improve internal linking as new public pages appear.

## 9. Technical implementation spec

Framework:

- Next.js App Router.

Rationale:

- Strong metadata support.
- Built-in Open Graph image handling.
- `robots.txt` and `sitemap.xml` routes.
- Favicon conventions.
- Efficient image handling.
- Good fit for a performant, content-rich marketing site.

Implementation rules:

- Semantic HTML.
- Accessible contrast.
- Responsive layouts designed mobile-first.
- Performant images and lightweight animation.
- Clean component architecture with shared design tokens.
- Centralized metadata and content definitions to reduce drift.
